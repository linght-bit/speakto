## 📋 Проблемы которые были исправлены

### ❌ Проблема 1: gameConfig не загружалась
**Причина:** Файл создавал переменную `gameConfig` но не прикреплял её к `window`
**Решение:** Добавил `window.gameConfig = gameConfig;` в конце файла

### ❌ Проблема 2: Другие модули не были доступны глобально
**Причина:** Только `eventSystem` был на window, остальные модули оставались локальными
**Решение:** Добавил экспорт на window для всех модулей:
- `window.getGameState = getGameState;`
- `window.updateGameState = updateGameState;`
- `window.resetGameState = resetGameState;`
- `window.initI18n = initI18n;`
- `window.getText = getText;`
- И т.д.

### ❌ Проблема 3: bootstrap.js требовал всех JSON файлов
**Причина:** Если хоть один fetch не загружался, вся инициализация падала
**Решение:** Переработал bootstrap.js:
- Используем try-catch для каждого fetch
- Если файл не загружается, используем дефолтные значения
- Игра запускается даже без данных (показывает "Game Loading...")

### ❌ Проблема 4: Системы обращались к getGameState напрямую
**Причина:** `window.getGameState` мог быть undefined на момент вызова
**Решение:** Добавил проверки `window.getGameState?.()` и try-catch в системах

### ❌ Проблема 5: GameRenderer падал если canvas не существовал
**Причина:** Код предполагал что canvas всегда есть
**Решение:** Добавил проверки и обработку ошибок:
```javascript
if (this.canvas) {
  this.ctx = this.canvas.getContext('2d');
}
if (!this.ctx) {
  console.warn('Canvas недоступен');
  return;
}
```

### ❌ Проблема 6: getText обращался к getGameState без проверки
**Причина:** На ранних стадиях загрузки getGameState может быть undefined
**Решение:** Обернул в try-catch с дефолтным языком 'ru'
