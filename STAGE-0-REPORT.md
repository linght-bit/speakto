### ✅ 5. Архитектурные принципы установлены

#### Правило 1: Один источник правды
```javascript
// gameState.js — ЕДИНЫЙ источник всех данных о состоянии
getGameState() → вся информация о игре
updateGameState() → единственный способ изменения
```

#### Правило 2: Тексты только в i18n
```javascript
// Запрещено
'Привет!' // ❌ Хардкод

// Разрешено
getText('dialogue.hello') // ✓ Ключ
```

#### Правило 3: Данные только в /data
```javascript
// Запрещено
const quests = [{id: 1, title: '...'}] // ❌ Хардкод

// Разрешено
const quests = await fetch('data/quests.json') // ✓ Data-driven
```

#### Правило 4: События для связи
```javascript
// Запрещено
questSystem.completeQuest(); // ❌ Прямой вызов

// Разрешено
eventSystem.emit('quest:completed') // ✓ События
eventSystem.on('quest:completed', handler) // Слушаем
```

## 🎮 Что происходит при загрузке игры

```
1. Браузер загружает index.html
   ↓
2. Загружаются модули (в правильном порядке):
   - events.js (система событий)
   - gameConfig.js (конфигурация)
   - gameState.js (состояние)
   - getText.js (i18n)
   - systems/* (системы)
   - ui/render.js (рендеринг)
   ↓
3. bootstrap.js запускает initGame():
   - Загружает JSON файлы (fetch)
   - Инициализирует i18n
   - Загружает данные в системы
   - Сбрасывает gameState на дефолт
   - Запускает gameRenderer.render()
   ↓
4. gameRenderer рисует на canvas
   - Каждый фрейм (requestAnimationFrame)
   - Использует данные из gameState
   - Использует текты из i18n
   ↓
5. test-loader.js проверяет загрузку (консоль)
   ✅ Все модули готовы!
```
